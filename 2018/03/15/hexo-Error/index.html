Hexo部署成功后全没显示新文章？

发表于 2018-03-15

其实之前这个问题我也会遇到过，那时候是在Ubuntu系统下，只懂得说

部署不成功？

本地执行预览命令，发现没有问题，然后执行

也没有报错，只是出现一堆警告而已，最后修改是已经提交到git remote仓库的, INFO Deploy done: git 也说明了部署操作成功了。
那么为什么会出现这个问题呢？

排查问题

上github查看chenpt.github.io这个repo，里面已经有最新的文章了。
我把本地预览时的端口号后面的路径复制，前面加上博客绑定的域名，发现可以访问这篇文章，chenpt.cc/2018/03/13/title/#more，可是直接进入chenpt.cc 首页没有显示文章，我想会不会是缓存的问题，我将地址发到微信上，用电脑微信打开，发现一切正常，文章也出现了，转过头打开控制台把chenpt.cc的缓存全清了，再部署一遍

原因


打开控制台一看NetWork，发现进入网站所请求的资源都是来自浏览器缓存，
状态码都为200，from disk cache or from memory cache，所以这说明了，所有东西都是在客户端缓存请求的，实际上并没有发出http请求给服务器.（这里是github page）

谷歌了一下发现 github Page有一个默认的刷新时间的
